## 2. Command Coverage

| Command | Syntax / Key Options | Safety Behavior |
|---------|----------------------|-----------------|
| **list** | `ferrumctl admin tokens list [--actor-id <ID>] [--role <ROLE>] [--active-only] [--limit <N>] [--format text|json]` | Metadata only; secret values never returned; supports filtering by actor, role, active state, and page size |
| **create** | `ferrumctl admin tokens create --actor-id <ID> --role <ROLE> [--scope <SCOPE>]... [--description <TEXT>] [--expires-in-days <N>|--expires-at <TIMESTAMP>] [--json]` | Token value printed **once** at creation; JSON output supported; TTL validation enforced |
| **revoke** | `ferrumctl admin tokens revoke <TOKEN_ID> [--reason <TEXT>] [--force]` | Interactive confirmation unless `--force`; optional reason; audit log emitted |
| **rotate** | `ferrumctl admin tokens rotate <TOKEN_ID> [--expires-in-days <N>|--expires-at <TIMESTAMP>] [--reason <TEXT>] [--json] [--force]` | Revokes old token + creates new; expires options supported; optional reason; `--json` output; `--force` skips confirmation |
