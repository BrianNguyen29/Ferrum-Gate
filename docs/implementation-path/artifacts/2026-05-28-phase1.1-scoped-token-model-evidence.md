## Requirement-to-Evidence Table

| Requirement | Evidence Location | Notes |
|-------------|-------------------|-------|
| Token model structs, enums, roles, default scopes | `crates/ferrum-proto/src/token.rs` | `ScopedToken`, `TokenRole` (Admin, Operator, PolicyAuthor, Auditor, Agent, ReadOnly), `AuthMode::Scoped`, request/response DTOs (`CreateTokenRequest`, `CreateTokenResponse`, `ScopedTokenMeta`, `RevokeTokenRequest`, `RotateTokenRequest`, `TokenListResponse`). Default scopes per role defined in `TokenRole::default_scopes()`. |
| Scope list and HTTP path mapping | `crates/ferrum-gateway/src/server.rs` — `required_scope_for_path()` | Maps `(method, path)` to required scope. Covers: `intent:submit`, `proposal:evaluate`, `capability:mint`, `execution:authorize`, `execution:prepare`, `execution:execute`, `execution:verify`, `execution:compensate`, `approval:resolve`, `policy:read`, `policy:write`, `provenance:read`, `admin:tokens`, `admin:audit`. Fallback for unknown paths defaults to `admin:tokens` (deny-by-default behavior present). |
| Scoped auth middleware basics | `crates/ferrum-gateway/src/server.rs` — `scoped_auth_middleware()` | Validates token via lookup hash + salted hash (constant-time comparison), checks revocation, checks expiration, checks required scope against token scopes (supports wildcard `"*"`), updates `last_used_at` best-effort. |
| Token repo / storage abstraction | `crates/ferrum-store/src/repos.rs` — `TokenRepo` trait | Async trait: `insert`, `get`, `get_by_lookup_hash`, `list`, `revoke`, `touch`. Concrete implementations exist for SQLite and PostgreSQL. `StoreFacade::tokens()` provides runtime access. |
| Token hashing / salting | `crates/ferrum-gateway/src/server.rs` | `hash_token_value()` (blake3 raw value for lookup), `hash_token_with_salt()` (blake3(salt \|\| value) for verification), `generate_token_salt()` (16-byte random). Token value format: `fgt_<base64url(48 bytes)>`. |
| CLI / token lifecycle presence (supporting evidence only) | `bins/ferrumctl/src/main.rs` | `ferrumctl admin tokens list`, `create`, `revoke`, `rotate` commands exist. This supports Phase 1.1 but does **not** close Phase 1.4 (full CLI polish) or Phase 1.3 (full lifecycle evidence). |
