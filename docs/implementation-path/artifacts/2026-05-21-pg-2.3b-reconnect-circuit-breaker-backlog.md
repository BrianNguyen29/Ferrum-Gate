## Why deferred

1. `sqlx::PgPool` already provides transparent reconnect with exponential backoff. A local Docker restart test (2026-05-18) confirmed ferrumd recovers without restart.
2. PG-2.3a (acquire-timeout metrics + pool-saturation readiness) gives fail-closed behavior when the pool is exhausted.
3. A circuit breaker is only meaningful with a load-balancer or multi-node topology (PG-5 / HA). That is out of v1 scope.

## Trigger for revisit

- HA design (managed PG, Patroni, or load-balanced topology) where per-instance health state and concrete RTO need definition.
- Operator requirement for custom retry semantics (e.g., bounded retries with jitter, fallback to read-only mode).
