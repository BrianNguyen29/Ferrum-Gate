## Requirement-to-Evidence Table

| Requirement | Evidence Location | Notes |
|-------------|-------------------|-------|
| Create scoped token | `crates/ferrum-gateway/src/server.rs` — `create_token` | Creates token, validates TTL, generates lookup hash + salted verification hash, stores metadata, returns token value once. |
| Token value returned once / list redacted | `crates/ferrum-proto/src/token.rs`; `list_tokens` handler | `CreateTokenResponse` includes token value. `ScopedTokenMeta` / list response omits secret material (`token_hash`, lookup hash, salt). |
| Revoke token | `revoke_token` handler; `TokenRepo::revoke()` | Sets `revoked_at` / `revoked_reason`; revoked tokens fail scoped auth. |
| Rotate token | `rotate_token` handler | Revokes old token and creates a new token with same actor/role/scopes; records `rotated_from`; validates TTL. |
| `last_used_at` | `ScopedToken` model; SQLite/Postgres repos; auth middleware | `TokenRepo::touch()` updates `last_used_at` after successful scoped auth on best-effort path. |
| `expires_at` | `ScopedToken` model; create/rotate handlers; auth middleware | Create/rotate TTL max validation; expired tokens fail auth with 401. |
| Audit logging | `AuditAction::TokenCreate`, `TokenRevoke`, `TokenRotate`; handlers | Token lifecycle operations emit audit actions. |
| Storage / repo abstraction | `TokenRepo`; SQLite and PostgreSQL token repos | `insert`, `get`, `get_by_lookup_hash`, `list`, `revoke`, `touch` supported. |
