> **Scope**: Single-node SQLite only. No PostgreSQL/multi-node. No production-ready claim.
> **Constraint**: Do not sign G2, do not claim pilot accepted, do not execute on target host.

---

## Purpose

This document describes the two-phase handoff for Path 2:

1. **Phase A — Repo-Side Preparation** (this document): Repository artifacts are prepared,
   examples are provided, and documentation is hardened. This phase is complete when this
   document is finalized.

2. **Phase B — Target Execution & Signoff** (blocked): Operator takes possession of the
   prepared artifacts, deploys to target environment, runs drills, and signs G2 gates.
   This phase is blocked until Phase A is complete and the operator has provided the
   target environment details via doc 65.

**G2 gates are NOT complete when Phase A ends. They require operator execution on the
target host and explicit signoff.**

### B.2 Phase B Steps (Operator-Owned)

After all blockers are resolved:

1. **Adapt configs**: Copy `configs/ferrumgate.prod.toml` → `/etc/ferrumgate/ferrumgate.toml`;
   update `bind_addr`, `store.path`, `auth_mode`. Copy `nginx-ferrumgate.conf` and update
   `server_name`, TLS paths. Copy `ferrumd.env.example` → `/etc/ferrumgate/ferrumd.env`
   and insert generated token.

2. **Deploy systemd units**: Copy `ferrumd.service` and `ferrumgate-backup.service` to
   `/etc/systemd/system/`. Run `systemctl daemon-reload && systemctl enable --now ferrumd`.

3. **Verify probes**: Run Phase 2 probe sequence from doc 62 §2.2. Record outputs.

4. **Complete doc 65**: Fill all PROVIDE fields with actual target values.

5. **Complete doc 63**: Mark each field as PROVIDE, OPERATOR-GENERATED, or DERIVED DURING DEPLOY.

6. **Run compensation drills**: Execute D1–D6 drills per doc 62 §Phase 3. Capture output.

7. **Run restore drill**: Execute non-prod restore drill per doc 62 §Phase 5.

8. **Generate evidence skeletons**: Run `generate_evidence_skeleton.py` against drill outputs.

9. **Complete doc 59**: Fill G2.1–G2.8 evidence fields. Do NOT mark complete without actual evidence.

10. **Sign doc 54**: Operator reviews all evidence and signs the pilot acceptance statement.

### B.3 Phase B Completion Criteria

Phase B is complete when:
- All probes return expected results on the target host
- All compensation drills executed and results captured
- Restore drill successful with `PRAGMA integrity_check` passing
- Doc 59 G2.1–G2.8 filled with actual evidence
- Doc 54 signed by operator

**Only after Phase B is complete may the pilot be considered for limited production use.**
