## Prerequisites

- Rust toolchain (stable)
- `cargo` and `make`
- A POSIX shell
- `curl`

## Validated endpoint sequence

The following four endpoints have been validated locally with `auth=disabled` and in-memory SQLite. The remaining steps in the full quickstart are documented but NOT validated end-to-end.

| Step | Endpoint | Method | Expected status |
|------|----------|--------|-----------------|
| 1 | `/v1/healthz` | `GET` | HTTP 200 `{"status":"ok"}` |
| 2 | `/v1/intents/compile` | `POST` | HTTP 200 with sanitized `intent_id` |
| 3 | `/v1/proposals/{proposal_id}/evaluate` | `POST` | HTTP 200 decision `Allow` (may include advisory warning) |
| 4 | `/v1/capabilities/mint` | `POST` | HTTP 200 with lease |

## Quickstart flow

### 1. Build

```bash
cargo build --release
```

### 2. Start ferrumd locally

Use the dev config with loopback-only binding:

```bash
FERRUMD_BIND_ADDR=127.0.0.1:18080 ./target/release/ferrumd --config configs/ferrumgate.dev.toml
```

Expected log lines:

```text
auth_mode=disabled
store_dsn=sqlite::memory:
```

Server listens on `127.0.0.1:18080`.

### 3. Health check

```bash
curl http://127.0.0.1:18080/v1/healthz
```

Expected:

- HTTP status: `200`
- Response body: `{"status":"ok"}`

### 4. Submit intent

```bash
curl -X POST http://127.0.0.1:18080/v1/intents/compile \
  -H "Content-Type: application/json" \
  -d '{"principal_id":"a8d023fb-a81b-4bc1-8397-5ba921e8a233","title":"demo-write","goal":"write a demo file","agent_plan_summary":"write /tmp/ferrum-demo.txt","trusted_context":{},"raw_inputs":[],"requested_resource_scope":[],"metadata":{}}'
```

> In dev mode any UUID-shaped `principal_id` is accepted. Capture the `intent_id` from the response as `<INTENT_ID>`.
>
> Validated example response: sanitized `intent_id` `a8d023fb-a81b-4bc1-8397-5ba921e8a233`.

### 5. Evaluate proposal

> In a real flow the proposal is generated by an agent or client. The example below is a minimal stand-in.

```bash
curl -X POST http://127.0.0.1:18080/v1/proposals/6a8c9b36-f554-4bf1-957a-f3e06de84ca8/evaluate \
  -H "Content-Type: application/json" \
  -d '{"proposal_id":"6a8c9b36-f554-4bf1-957a-f3e06de84ca8","intent_id":"<INTENT_ID>","step_index":1,"title":"demo proposal","tool_name":"fs.write","server_name":"fs","raw_arguments":{"path":"/tmp/ferrum-demo.txt","content":"hello"},"expected_effect":"write file","estimated_risk":"Low","requested_rollback_class":"R0NativeReversible","taint_inputs":[],"metadata":{},"created_at":"2026-05-19T00:00:00Z"}'
```

Note the `<PROPOSAL_ID>` you used in the URL; you will need it for the next step.

> Validated example response: decision `Allow` with advisory warning `advisory mismatch: inferred effect FileMutation is not in allowed outcomes`.

### 6. Mint capability

```bash
curl -X POST http://127.0.0.1:18080/v1/capabilities/mint \
  -H "Content-Type: application/json" \
  -d '{"intent_id":"<INTENT_ID>","proposal_id":"<PROPOSAL_ID>","tool_binding":{"server_name":"fs","tool_name":"fs.write"},"resource_bindings":[{"kind":"File","path":"/tmp/ferrum-demo.txt","mode":"Write"}],"argument_constraints":[],"taint_budget":{"max_taint_score":0,"allow_external_tool_output":false,"allow_external_metadata":false,"allow_untrusted_text":false},"approval_binding":null,"requested_ttl_secs":300,"metadata":{}}'
```

Capture `<CAPABILITY_ID>` from the response.

> Validated example response: HTTP 200 with a capability lease.

## Extended flow (scaffold / not yet validated)

The steps below are documented for completeness but have **not** been validated end-to-end in the local quickstart run.

### 7. Authorize execution

```bash
curl -X POST http://127.0.0.1:18080/v1/executions/authorize \
  -H "Content-Type: application/json" \
  -d '{"proposal_id":"<PROPOSAL_ID>","capability_id":"<CAPABILITY_ID>","dry_run":false}'
```

Capture `<EXECUTION_ID>` from `execution.execution_id` in the response.

### 8. Prepare

```bash
curl -X POST http://127.0.0.1:18080/v1/executions/<EXECUTION_ID>/prepare \
  -H "Content-Type: application/json"
```

No request body is required.

### 9. Execute

```bash
curl -X POST http://127.0.0.1:18080/v1/executions/<EXECUTION_ID>/execute \
  -H "Content-Type: application/json" \
  -d '{"payload":{"content":"hello"}}'
```

### 10. Verify

```bash
curl -X POST http://127.0.0.1:18080/v1/executions/<EXECUTION_ID>/verify \
  -H "Content-Type: application/json"
```

No request body is required.

### 11. Query lineage

```bash
curl http://127.0.0.1:18080/v1/provenance/lineage/<EXECUTION_ID>
```
